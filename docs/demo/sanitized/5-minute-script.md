# Dream Studio 5-Minute Sanitized Demo Script

Audience: product, engineering, operations, and AI-platform evaluators.

Mode: public sanitized. Screenshots are synthetic and contain no private operator history.

## 0:00-0:30 - Opening

Narration:

> Dream Studio is a local-first AI orchestration and operational intelligence platform. It helps teams use AI tools without losing track of authority, evidence, approvals, validation, privacy, and release readiness.

Show: the Observe Mode screenshot.

## 0:30-1:45 - Observe Mode

Narration:

> First, Dream Studio observes the operating state. The dashboard is not the source of truth; it is a derived view over SQLite authority, Contract Atlas, telemetry, evidence references, security and production-readiness status, project authority, and module profiles.

Point to:

- project portfolio status;
- Contract Atlas freshness;
- validation status;
- security and readiness status;
- evidence-linked dashboard attention.

Demo line:

> The key distinction is that missing evidence is shown as missing, not converted into fake confidence.

## 1:45-3:00 - Assist Mode

Show: the Assist Mode screenshot.

Narration:

> Next, Dream Studio assists. It can generate a context packet for an AI adapter, import a manual or adapter result, preserve evidence refs, and attribute the work to the project, milestone, Work Order, skill, workflow, adapter, and validation outcome.

Point to:

- context packet generation;
- adapter result import;
- evidence capture;
- attribution;
- privacy boundary;
- cost and token honesty.

Demo line:

> Adapters are workers, not authority. Their results normalize back into Dream Studio records.

## 3:00-4:30 - Operate Mode

Show: the Operate Mode screenshot.

Narration:

> Finally, Dream Studio operates through guardrails. A risky continuation becomes a route decision and Work Order. The policy engine classifies the action, validation runs in an isolated path, approval-required items appear in attention, and destructive actions remain blocked until explicitly approved.

Point to:

- route decision;
- Work Order scope;
- validation result;
- approval boundary;
- release readiness impact;
- safe next action.

Demo line:

> Dream Studio is designed to prevent hidden prompt-chaining from turning into accidental cleanup, release, deploy, or private-data exposure.

## 4:30-5:00 - Close

Narration:

> The result is an installed local platform for AI work: observable, evidence-backed, privacy-safe, adapter-aware, and release-gated. The public demo is sanitized; live operator state remains private.

Close with:

> Dream Studio is public-demo-ready in sanitized mode and private-demo-ready for live local walkthroughs.
