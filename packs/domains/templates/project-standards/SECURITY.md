# Security Policy — {{project_name}}

## Reporting a Vulnerability

Email **{{contact_email}}** with subject `[{{project_name}}] Security Report`.

We aim to respond within **30 days**.

## What to Report

- Credential or token exposure in hooks or config files
- Hook injection vulnerabilities
- Path traversal in skill or script runners
- Dependency CVEs in requirements files

## What NOT to Report

General bugs belong in GitHub Issues.
