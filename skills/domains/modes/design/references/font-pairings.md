## Usage Tips

1. **Mobile Optimization:** Pairings #58-#75 are specifically designed for mobile interfaces with consideration for screen size, touch targets, and native app conventions.

2. **Loading Strategy:** For web apps, use CSS `@import` or `<link>` tags. For React Native/Expo, use `expo-font` and `useFonts()` hook.

3. **Weight Hierarchy:** Most pairings recommend specific weight ranges. Heavy weights (700-900) for headings, medium (500-600) for subheadings, regular (400) for body.

4. **Letter Spacing:** Display fonts often benefit from negative letter spacing (-0.5 to -1.5px), while labels/buttons benefit from positive spacing (+1 to +2px).

5. **Line Height:** Headlines typically use 1.0-1.2, body text uses 1.4-1.6 for optimal readability.

6. **Accessibility:** Pairings #16, #30, #41, #48 prioritize accessibility and WCAG compliance.

7. **Fallback Fonts:** Always include system fallbacks: `-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`
