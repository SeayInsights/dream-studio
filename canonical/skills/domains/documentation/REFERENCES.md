# Documentation References

## Technical Writing

### awesome-docs (824 stars)
**URL**: https://github.com/testthedocs/awesome-docs

Documentation best practices including:
- Diátaxis framework (tutorials, how-to, reference, explanation)
- Style guides (Google, Microsoft, Write the Docs)
- Documentation tooling and generators
- API documentation patterns (OpenAPI, examples, error codes)

**Why this matters**: Good documentation is force multiplier for adoption and maintenance. Diátaxis provides a proven structure for organizing docs by user intent.

## Style Guides

### Google Developer Documentation Style Guide
**URL**: https://developers.google.com/style

Active voice, present tense, second person, short sentences.

### Microsoft Writing Style Guide
**URL**: https://learn.microsoft.com/en-us/style-guide/

Conversational tone, accessibility, inclusive language.

## Documentation Frameworks

### Diátaxis
**URL**: https://diataxis.fr/

Four quadrants: tutorials (learning), how-to (tasks), reference (information), explanation (understanding).
