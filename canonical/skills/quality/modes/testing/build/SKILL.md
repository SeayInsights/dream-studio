# Testing Build Mode

## Metadata
- **Pack:** quality
- **Mode:** testing:build
- **Type:** generative guidance
- **Model:** sonnet
- **Inputs:** target_file, target_function, context_mode
- **Outputs:** test_structure_guidance, test_stubs

## Before you start
1. Read `../gotchas.yml` — contains patterns to recognize as positive or avoid.
2. Read `../rules.yml` — build mode checks apply rules where `action.build_mode` is not null.
3. Read `../config.yml` — check critical_path_globs and halt_on_critical settings.

## Trigger
`ds-quality:testing:build`, `generate tests:`, `write tests:`, `add test:`

## Purpose
Provide structured guidance for writing tests that follow approved patterns. Runs static checks on the target code first, then generates test stubs or guidance. Halts on critical findings in existing test files (config: `halt_on_critical_in_build: true`).

---

## Step 1 — Pre-generation Check

Run the static-only rules on any EXISTING test file for the target module:
- tst-011: No sleep() already present?
- tst-007: Existing test names are behavior-describing?
- tst-015: No slow patterns in existing unit tests?

If critical finding in existing test file AND `config.halt_on_critical_in_build: true`:
```
⛔ Build mode halted: {rule_id} critical finding in existing test file {filename}.
   Fix: {rule.remediation.summary}
   Use ds-quality:testing:audit to see all findings before adding more tests.
```

---

## Step 2 — Analyze the Code Under Test

For the target function/module:
1. Identify the public API surface (what can be invoked)
2. Identify the critical paths (functions matching critical_path_globs)
3. Identify external dependencies (DB connections, API calls, file I/O) — these are boundary mock candidates
4. Identify inputs and observable outputs/side effects
5. Flag any untestable patterns (hidden global state, hardcoded non-injectable dependencies)

If untestable patterns exist:
```
⚠ Testability concern: {description}
   Consider refactoring before writing tests, or document why this pattern is intentional.
   See ds-quality:code-quality for code testability rules.
```

---

## Step 3 — Generate Test Structure Guidance

Provide guidance following the approved patterns:

### Naming
Use the pattern: `test_<behavior>_when_<condition>_then_<expected_result>`
- `test_invoice_total_includes_tax`
- `test_user_creation_fails_when_email_is_duplicate`
- `test_migration_applies_cleanly_on_empty_db`

Avoid: `test_create_user`, `test_invoice`, `test_it`

### Structure (AAA)
```python
def test_{behavior}_{condition}():
    # Arrange: set up test data, fixtures, mocks
    {setup}

    # Act: invoke the code under test (ONE call)
    result = {code_under_test}({inputs})

    # Assert: verify observable result (one concept)
    assert {expected_outcome}
```

### Mocking guidance
Mock at boundaries:
```python
# Good: mock the external dependency (DB, API)
@patch('module.external_api_client')
def test_service_handles_api_failure(mock_api):
    mock_api.get.side_effect = ConnectionError()
    result = service.fetch_data()
    assert result is None

# Bad: mock an internal helper
service._validate_input = MagicMock(return_value=True)  # don't do this
```

### Coverage guidance
For the target module:
- Start with the critical path (functions matching critical_path_globs) → integration test first
- Then unit tests for individual functions
- Cover: happy path, edge cases, error paths

### Dream Studio-specific pytest patterns
```python
# DB-isolated test using Dream Studio's fixture pattern
def test_something(tmp_path):
    # conftest.py already redirected DREAM_STUDIO_DB_PATH at import time
    # tmp_path gives a fresh temp dir per test
    db = tmp_path / "test.db"
    conn = sqlite3.connect(str(db))
    # ... test logic

# Parametrized test
@pytest.mark.parametrize("input,expected", [
    (1, "one"),
    (2, "two"),
])
def test_number_to_word_for_known_values(input, expected):
    assert number_to_word(input) == expected

# Monkeypatching for determinism
def test_timestamp_is_recorded(monkeypatch, tmp_path):
    fixed_time = datetime(2026, 1, 1, 12, 0, 0)
    monkeypatch.setattr("module.datetime", MockDateTime(fixed_time))
    record = create_record()
    assert record.created_at == fixed_time
```

## Step 4 — Generate Test Stubs

Generate test stubs for the target function:

```python
# Auto-generated test stubs for {module_name}.{function_name}
# Generated by ds-quality:testing:build
# Apply the AAA pattern: Arrange / Act / Assert

import pytest
{relevant_imports}

{if_has_fixtures}
@pytest.fixture
def {module_name}_fixture():
    # TODO: set up test state
    pass


def test_{function_name}_returns_{expected_for_happy_path}():
    # Arrange
    {setup_for_happy_path}

    # Act
    result = {function_name}({inputs})

    # Assert
    assert {expected_outcome}


def test_{function_name}_handles_{error_case}():
    # Arrange
    {setup_for_error_case}

    # Act + Assert (for exception tests)
    with pytest.raises({ExceptionType}):
        {function_name}({invalid_inputs})
{if_critical_path}

# Integration test stub (this function is in a critical path)
def test_{function_name}_integration_{end_to_end_description}():
    # Arrange: set up full stack (real DB, real dependencies)
    # Act: invoke from the entry point
    # Assert: verify observable end-to-end result
    pass
```
