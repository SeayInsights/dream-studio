# Implementation Plan: Grade-A Upgrade — Memory, CI, Token Efficiency

**Date**: 2026-04-29 | **Spec**: `.planning/specs/grade-a-upgrade/spec.md`

## Summary

Four independent deliverables to bring Memory/Persistence (C→A), CI Gate (D→A),
Token Efficiency (Unknown→A), and First-Run Setup (gap→A) to grade-A.
P0 Setup ships as a standalone PR first; the three upgrades ship as sequenced commits
in a second PR.

## Technical Context

**Language/Version**: Python 3.11+ (CI matrix: 3.10–3.12; setup.py requires ≥3.11)
**Primary Dependencies**: stdlib only — `sqlite3` (FTS5), `subprocess` (ci_gate), `pathlib`, `json`; **zero new pip packages**
**Storage**: SQLite FTS5 db at `{memory_dir}/memory.db`; token-log.md (existing, extended); token-benchmark.md (new)
**Testing**: pytest (existing `tests/unit/` + `tests/integration/`; coverage via pyproject.toml)
**Target Platform**: Claude Code CLI, cross-platform (Windows/Mac/Linux)
**Project Type**: CLI plugin / hook system
**Performance Goals**: Memory index rebuild <100ms on 500 entries (FR-M06); ci-gate run <60s (SC-C02)
**Constraints**: Zero new pip dependencies; hook chain must not block user prompt; settings.json merge is additive-only

## Constitution Check

- No new external dependencies — FTS5 is compiled into CPython by default since 3.8
- All hooks are additive — existing UserPromptSubmit chain preserved; on-memory-retrieve inserted, not replacing
- P0 setup is non-destructive — settings.json merge only adds missing keys, never overwrites
- CI gate LLM review is advisory (non-blocking) — won't break CI if ANTHROPIC_API_KEY is absent
- PRs scoped to <120 lines each — enforced by splitting P0 from the 3 upgrades

## Risks & Mitigations

| Risk | Impact | Mitigation |
|---|---|---|
| UserPromptSubmit hook output may not inject as context | High | Validate on-memory-retrieve output format against on-pulse before wiring hooks.json; test with real session |
| FTS5 not available (unusual Python build) | Low | Add `sqlite3.connect(':memory:').execute('CREATE VIRTUAL TABLE t USING fts5(x)')` check in module init; fail gracefully |
| ci-gate adds >60s to CI | Medium | Reuse existing pytest/lint runs; don't re-run, just orchestrate the same make targets |
| settings.json merge corrupts user config | High | Read → deep merge (additive only) → write; preserve all existing keys; note: .claude/ settings not .git/ |
| Makefile `ci-gate` and `setup` targets in two PRs | Low | P0 merges first; grade-a-upgrade branches from updated main — no conflict |
